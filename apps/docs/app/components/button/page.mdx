# Button

A foundational interactive component designed to trigger actions and interactions. Built with accessibility in mind, supporting keyboard navigation, focus states, and semantic HTML.

The Button component adapts to your theme's accent color by default, providing consistent visual hierarchy across different variants while maintaining optimal contrast and readability.

<CodePreview code={`import { Button } from '@kushagradhawan/kookie-ui';

<Button>Button</Button>`}>
  <div style={{ display: 'inline-block' }}>
    <Button>Button</Button>
  </div>
</CodePreview>

## API Reference

This component is based on the `button` element and supports common margin props.

| Prop         | Type                                                                            | Default   | Description              |
| ------------ | ------------------------------------------------------------------------------- | --------- | ------------------------ |
| size         | `"1"` \| `"2"` \| `"3"` \| `"4"`                                                | `"2"`     | Size of the button       |
| variant      | `"classic"` \| `"solid"` \| `"soft"` \| `"surface"` \| `"outline"` \| `"ghost"` | `"solid"` | Visual style variant     |
| color        | `string`                                                                        | -         | Accent color             |
| highContrast | `boolean`                                                                       | `false`   | Increases color contrast |
| radius       | `"none"` \| `"small"` \| `"medium"` \| `"large"` \| `"full"`                    | -         | Border radius            |
| loading      | `boolean`                                                                       | `false`   | Shows loading spinner    |
| fullWidth    | `boolean`                                                                       | `false`   | Makes button full width  |

Use the `size` prop to control the button's dimensions and font size. Sizes are designed to maintain consistent visual hierarchy and work well together in interfaces.

### Color

By default, buttons use your theme's accent color. Use the `color` prop to override this when you need semantic meaning or specific branding. This is especially useful for destructive actions (red), success states (green), or warning actions (orange).

  <Button color="crimson" variant="soft">Delete</Button>
</Flex>`}
>

<Button color="cyan" variant="soft">Preview</Button>

Use the `loading` prop to show a spinner while async operations are in progress. The button automatically becomes disabled and maintains its original dimensions to prevent layout shift. The spinner inherits the button's color and size.

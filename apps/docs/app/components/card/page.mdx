# Card

A foundational container component designed to group related content and provide visual separation. Built with semantic HTML and supports various visual styles to match different design contexts.

The Card component provides a consistent way to present content with proper spacing, elevation, and visual hierarchy across your application.

<CodePreview code={`import { Card } from '@kushagradhawan/kookie-ui';

<Card>
  {/* Your content goes here */}
</Card>`}>
  <Card style={{ width: '200px', height: '100px' }} />
</CodePreview>

## API Reference

This component is based on the `div` element and supports common margin props.

| Prop    | Type                                           | Default     | Description              |
| ------- | ---------------------------------------------- | ----------- | ------------------------ |
| size    | `"1"` \| `"2"` \| `"3"` \| `"4"` \| `"5"`      | `"1"`       | Size and padding of card |
| variant | `"surface"` \| `"classic"` \| `"ghost"` \| `"soft"` | `"surface"` | Visual style variant     |

### Variant

Each variant provides different visual emphasis and interaction styles:

- **Surface**: Default variant with border and subtle elevation
- **Classic**: Elevated appearance with gradient background and shadow
- **Ghost**: Minimal variant that inherits parent spacing, ideal for list items
- **Soft**: New minimal variant with subtle background tint, no borders or shadows

### Soft Variant (New in Kookie UI)

The `soft` variant is a new addition to Kookie UI that provides a minimal appearance with just a subtle background tint. Unlike other variants, it has no borders, shadows, or elevated appearance, making it perfect for subtle content grouping.

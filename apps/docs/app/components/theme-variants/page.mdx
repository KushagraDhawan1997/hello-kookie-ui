# Theme Variants

Kookie UI features a **comprehensive variant system** that provides consistent, adaptive styling across all components. This system maintains API compatibility with Radix Themes while implementing a systematic variant architecture for modern UI design.

## Variant Types Overview

Our design system supports **6 core variant types** with specific use cases:

### 1. Classic Variant - Traditional Depth
Traditional UI with depth, shadows, and gradients for professional applications.

### 2. Solid Variant - High Contrast  
High-contrast, prominent elements for primary actions and call-to-action buttons.

### 3. Soft Variant - Branded Subtlety
Subtle branded styling with accent colors for secondary actions and themed components.

### 4. Surface Variant - Neutral Surfaces
Neutral surface styling for containers, forms, and professional non-branded appearance.

### 5. Outline Variant - Minimal Borders
Minimal styling with border-only design for secondary buttons and clean form inputs.

### 6. Ghost Variant - Ultra Minimal
Invisible until interaction, perfect for floating labels and ultra-clean interfaces.

## Panel Background System

The **core innovation** of our variant system is automatic adaptation to panel background settings.

### Theme-Level Control

```jsx
<Theme panelBackground="solid">      {/* All components use solid colors */}
<Theme panelBackground="translucent"> {/* All components use alpha colors + blur */}
```

## Component Categories

Components are organized into **three surface categories** based on their color system:

### Interactive Components (Accent Colors)
- **Button, IconButton**: All variants (classic, solid, soft, surface, outline, ghost)
- **Badge**: Accent-based variants  
- **Switch**: Accent-based styling

### Container Components (Panel Colors)
- **Card**: Uses neutral panel color system
- **Sidebar**: Panel-based container surfaces
- **Dialog, Popover**: Neutral panel surfaces

### Form Components (Surface Colors)
- **TextField, TextArea, Select**: Form-specific surface system
- **Checkbox, Radio**: Form control surfaces
- **CheckboxCards, RadioCards**: Form surface styling

## Panel Background Benefits

### Solid Panel Background
- **Better contrast** with solid color scales
- **Improved readability** on opaque backgrounds  
- **Consistent brand presence** without transparency artifacts
- **Content-focused** interfaces

### Translucent Panel Background
- **Modern glass effects** with alpha transparency
- **Backdrop blur compatibility** for advanced UI aesthetics
- **Layered visual hierarchy** through transparency
- **Contemporary design** aesthetics

## Implementation Details

### Automatic Color Adaptation

Components automatically switch between color systems:

```css
.rt-Component:where(.rt-variant-soft) {
  /* Base: solid accent for solid panels */
  background-color: var(--accent-3);

  /* Auto-switch to alpha for translucent panels */
  :where([data-panel-background='translucent']) & {
    background-color: var(--accent-a3);
    backdrop-filter: var(--backdrop-filter-components);
  }
}
```

### Backdrop Filter System

- **`--backdrop-filter-panel`**: Strong blur (24px) for containers
- **`--backdrop-filter-components`**: Subtle blur (12px) for interactive elements
- **Automatic disabling**: Both become `none` on solid panels

### Surface Token Systems

The system uses three token systems:

1. **Accent colors**: `accent-2` → `accent-a2` for themed components
2. **Panel colors**: `--color-panel` for neutral containers  
3. **Surface colors**: `--color-surface` for form inputs

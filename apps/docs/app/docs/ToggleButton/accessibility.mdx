# Accessibility

ToggleButton accessibility extends Button with specialized requirements for toggle interactions. It provides proper ARIA attributes for toggle states, enhanced screen reader support for on/off announcements, and maintains all Button accessibility features.

For comprehensive documentation on shared accessibility features, see the **[Button Accessibility documentation](../Button/accessibility)**.

## Color and Contrast

ToggleButton uses the same color and contrast system as Button. For comprehensive documentation on color accessibility and high contrast support, see the **[Button Accessibility documentation](../Button/accessibility)**.

## Keyboard Navigation

ToggleButton uses the same keyboard navigation system as Button. For comprehensive documentation on keyboard accessibility, see the **[Button Accessibility documentation](../Button/accessibility)**.

## Screen Reader Support

ToggleButton extends Button's screen reader support with specialized toggle state announcements and ARIA attributes for on/off interactions.

### Toggle State Announcements

ToggleButton uses proper ARIA attributes to announce toggle states to screen readers. The `aria-pressed` attribute communicates the current on/off state.

<CodeBlock
  preview={
    <Flex gap="3" wrap="wrap">
      <ToggleButton size="3" variant="solid" pressed={true}>
        Enable Feature
      </ToggleButton>
      <ToggleButton size="3" variant="solid" pressed={false}>
        Disable Feature

```tsx
{/* Screen readers announce "Enable Feature, pressed" */}
<ToggleButton size="3" variant="solid" pressed={true}>
  Enable Feature
</ToggleButton>

{/* Screen readers announce "Disable Feature, not pressed" */}
<ToggleButton size="3" variant="solid" pressed={false}>
  Disable Feature
</ToggleButton>
```

</CodeBlock>

**Toggle State Communication:**
- **Pressed state:** Screen readers announce "pressed" to indicate the toggle is on
- **Unpressed state:** Screen readers announce "not pressed" to indicate the toggle is off
- **State changes:** Screen readers announce the new state when toggled

### Toggle-Specific Testing

When testing ToggleButton accessibility, ensure:

1. **Toggle State Announcements:** Screen readers announce "pressed" and "not pressed" states correctly
2. **State Changes:** Screen readers announce state changes when toggled
3. **ARIA Attributes:** aria-pressed attribute is properly managed
4. **Keyboard Toggle:** Toggle state changes with keyboard interaction (Space/Enter)
5. **Visual Feedback:** Pressed state provides clear visual indication

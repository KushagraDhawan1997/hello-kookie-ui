# Changelog

All notable changes to the ToggleButton component will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- Controlled and uncontrolled state handling through `pressed` and `onPressedChange` props
- Default state configuration with `defaultPressed` prop
- Automatic ARIA attribute management for accessibility compliance
- Clear visual feedback for on/off states across all variants
- Enhanced state change animations and transitions

### Changed
- Systematic visual treatment for pressed states across all variants
- Consistent state indication that works with existing Button design tokens
- Better integration with overall design system color and contrast
- Enhanced hover and focus states that work with toggle functionality

### Fixed
- Automatic `aria-pressed` attribute management based on toggle state
- Screen reader announcements for "pressed" and "not pressed" states
- Enhanced keyboard navigation support for toggle interactions
- Seamless integration with Button component through asChild pattern

## [1.0.0] - 2024-01-15

### Added
- Initial release of ToggleButton component
- Toggle state functionality with proper accessibility
- Four size options: 1, 2, 3, 4
- Six visual variants: classic, solid, soft, outline, surface, ghost
- Comprehensive color system with semantic options
- Full accessibility support with keyboard navigation
- Loading states with visual feedback
- Disabled states with proper ARIA attributes
- Polymorphic rendering with `as` prop
- High contrast mode support
- Responsive design with breakpoint support

### Technical
- Built on Radix UI Toggle primitive for robust state management
- CSS custom properties for theming
- TypeScript support with comprehensive type definitions
- Design token integration for consistent spacing and sizing

# Guidelines

ToggleButtons are state switches. When someone sees a toggle button, they instantly understand it can be turned on and off. Your job as a designer is to make the current state clear and guide people toward successful state changes.

This guide teaches you the psychological principles, visual hierarchy, and design decisions that create toggle button interfaces people trust and understand intuitively.

## Toggle State Design

Toggle buttons require clear visual communication of their current state. Users must instantly understand whether a toggle is on or off, and what will happen when they interact with it.

### Visual State Indicators

Toggle buttons use visual cues to communicate their current state. The pressed state should be immediately recognizable through color, styling, or other visual treatments.

<CodeBlock
  preview={
    <Flex gap="2" align="center">
      <ToggleButton size="3" variant="soft" pressed={true}>
        Dark Mode
      </ToggleButton>
      <ToggleButton size="3" variant="soft" pressed={false}>
        Light Mode
      </ToggleButton>
    </Flex>
  }
>

```tsx
{/* Clear visual indication of pressed state */}
<ToggleButton size="3" variant="soft" pressed={true}>
  Dark Mode
</ToggleButton>

<ToggleButton size="3" variant="soft" pressed={false}>
  Light Mode
</ToggleButton>
```

</CodeBlock>

**State Communication Principles:**
- **Pressed state:** Should feel "active" or "engaged" through visual treatment
- **Unpressed state:** Should feel "inactive" or "available" but not engaged
- **State changes:** Should provide immediate visual feedback when toggled
- **Consistency:** Same visual treatment should mean same state across all toggles

### State Labels

Toggle button labels should clearly communicate what the current state is and what will happen when toggled.

<CodeBlock
  preview={
    <Flex gap="2" align="center">
      <ToggleButton size="3" variant="outline" pressed={true}>
        Enabled
      </ToggleButton>
      <ToggleButton size="3" variant="outline" pressed={false}>
        Disabled
      </ToggleButton>
    </Flex>
  }
>

```tsx
{/* Clear state labels */}
<ToggleButton size="3" variant="outline" pressed={true}>
  Enabled
</ToggleButton>

<ToggleButton size="3" variant="outline" pressed={false}>
  Disabled
</ToggleButton>
```

</CodeBlock>

**Labeling Best Practices:**
- **Current state:** Label should describe the current state ("Enabled", "Dark Mode")
- **Action clarity:** Users should understand what will happen when toggled
- **Consistency:** Use consistent terminology across related toggles
- **Context:** Labels should make sense in the current interface context

## Common Mistakes

### Unclear State Indication

Using toggle buttons without clear visual indication of their current state.

<CodeBlock
  preview={
    <Flex gap="2" align="center">
      <ToggleButton size="3" variant="soft" pressed={true}>
        Toggle
      </ToggleButton>
      <ToggleButton size="3" variant="soft" pressed={false}>
        Toggle
      </ToggleButton>
    </Flex>
  }
>

```tsx
{/* ❌ Avoid: Unclear what state this represents */}
<ToggleButton size="3" variant="soft" pressed={true}>
  Toggle
</ToggleButton>

{/* ✅ Better: Clear state indication */}
<ToggleButton size="3" variant="soft" pressed={true}>
  Enabled
</ToggleButton>
```

</CodeBlock>

### Inconsistent State Labels

Using different terminology for the same toggle states across an interface.

<CodeBlock
  preview={
    <Flex gap="2" align="center">
      <ToggleButton size="3" variant="outline" pressed={true}>
        On
      </ToggleButton>
      <ToggleButton size="3" variant="outline" pressed={false}>
        Disabled
      </ToggleButton>
    </Flex>
  }
>

```tsx
{/* ❌ Avoid: Inconsistent terminology */}
<ToggleButton size="3" variant="outline" pressed={true}>
  On
</ToggleButton>
<ToggleButton size="3" variant="outline" pressed={false}>
  Disabled
</ToggleButton>

{/* ✅ Better: Consistent terminology */}
<ToggleButton size="3" variant="outline" pressed={true}>
  Enabled
</ToggleButton>
<ToggleButton size="3" variant="outline" pressed={false}>
  Disabled
</ToggleButton>
```

</CodeBlock>

### Missing Visual Feedback

Toggle buttons that don't provide clear visual feedback for state changes.

<CodeBlock
  preview={
    <Flex gap="2" align="center">
      <ToggleButton size="3" variant="ghost" pressed={true}>
        Feature
      </ToggleButton>
      <ToggleButton size="3" variant="ghost" pressed={false}>
        Feature
      </ToggleButton>
    </Flex>
  }
>

```tsx
{/* ❌ Avoid: No visual difference between states */}
<ToggleButton size="3" variant="ghost" pressed={true}>
  Feature
</ToggleButton>

{/* ✅ Better: Clear visual state indication */}
<ToggleButton size="3" variant="soft" pressed={true}>
  Feature Enabled
</ToggleButton>
```

</CodeBlock>
